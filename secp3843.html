Tutorials

End-to-End MS Azure Data Engineering Pipeline: Smart City

Figure 1.0: Dashboard built in Microsoft Power BI, connected to Azure Synapse Analytics Serverless SQL views for the Smart City ETL Pipeline

Project Overview: Developed a cloud-based, end-to-end data engineering pipeline to process and analyze a Smart City operational dataset. The objective was to migrate data from a local simulation environment into a scalable, query-optimized lakehouse data model.

  • Extraction & Transformation: Utilized **Azure Data Factory** to ingest raw data from an on-premises SQL Server database (`.\SQLEXPRESS`), orchestrating incremental data loading across the **Medallion Architecture** (Bronze, Silver, and Gold layers) using **Azure Databricks**.
  • Infrastructure & Security: Implemented security baselines directly within **Azure Data Lake Storage Gen2** by utilizing **Azure Managed Identities** and direct role assignments to bypass environment access restrictions and secure pipeline execution.
  • Visualization: Connected **Azure Synapse Analytics Serverless SQL** views to **Power BI Desktop**, implementing complex data modeling schemas to provide interactive analytics, tracking dynamic catalog shifts such as live inventory scaling.

Reference: Cloud infrastructure and pipeline architecture inspired by the guide Azure Data Engineering End-to-End Project by Kahan Data Solutions.

Apache Spark ETL Pipeline: Censo Escolar

Figure 1.0: Dashboard built in Metabase, connected to PostgreSQL Gold Layer (Star Schema) for Censo Escolar ETL Pipeline

Project Overview: Developed an end-to-end ETL (Extract, Transform, Load) pipeline to process and analyze the Brazilian School Census dataset. The objective was to transform a large-scale, flat CSV file into a query-optimized multidimensional data model.

  • Extraction & Transformation: Utilized PySpark to ingest the raw data, generate surrogate primary keys, and dynamically construct a Star Schema consisting of a central Fact table and 9 descriptive Dimension tables.
  • Infrastructure & Loading: Containerized the local architecture using Docker and loaded the transformed schema into a PostgreSQL data warehouse.
  • Visualization: Connected Metabase to the relational database to serve interactive business intelligence dashboards and validate the data model.

Reference: Pipeline architecture inspired by the guide Creating a Simple ETL Pipeline With Apache Spark.

Image Classification Using Convolutional Neural Network (CNN)

Figure 1.0: CNN model architecture for image classification trained on the CIFAR-10 dataset, including baseline and enhanced model evaluation.

Project Overview: Developed and evaluated an image classification pipeline using Convolutional Neural Networks (CNN) in Python. The objective was to build a baseline CNN model, identify its weaknesses, and implement enhancements to improve classification performance on a multi-class dataset.

  • Dataset & Preprocessing: Utilized the CIFAR-10 dataset comprising 60,000 colour images across 10 classes, applying normalization and train-test splitting for model readiness.
  • Baseline CNN Model: Constructed a sequential CNN architecture using TensorFlow/Keras with convolutional, pooling, and dense layers, trained using the SGD optimizer and evaluated on accuracy and loss metrics.
  • Model Enhancement: Identified weaknesses in the baseline model and applied improvements such as additional layers, activation tuning, and data augmentation to produce an enhanced CNN with comparative performance gains.
  • Evaluation & Comparison: Assessed both models using accuracy, loss curves, and confusion matrices to derive insights on classification performance across all 10 image categories.

Reference: Implementation guided by the tutorial Image Classification Using CNN

Solution Development for Product Packaging Manufacturer: PT. MPM Case Study

Figure 1.0: Data architecture and Star Schema design for PT. MPM case study, including core entities and their relationships across the ERP and BI pipeline.

Project Overview: Analyzed a published Business Intelligence (BI) case study on PT. MPM, a regional product packaging manufacturer in Indonesia. The objective was to derive the core problem and solution framing, and design supporting system artifacts including an Entity Relationship Diagram (ERD), Use Case Diagram, data architecture, and multidimensional Star Schema.

  • Problem Identification: PT. MPM lacked a centralized system to measure and monitor the performance of its core operations, causing top management to make decisions ineffectively and reducing its competitiveness against rivals.
  • Solution Design: Proposed a Business Intelligence Dashboard developed using the Waterfall methodology (Analysis → Requirement → Design → Development → Testing → Implementation) and built on Google Data Studio, comprising a Sales Performance Dashboard and a Production Performance Dashboard.
  • ERD Design: Derived an Entity Relationship Diagram from the DFD Level 0 of the paper, mapping six core entities — technical_ind_design, manufacturing, inventory_warehouse, sales, data_analyst, and top_management — and their data flow relationships across the ERP and BI pipeline.
  • ETL Pipeline: Data from ERP source systems underwent Extract, Transform, Load (ETL) processing via Google Sheets as an intermediate data warehouse before being loaded into Google Data Studio for dashboard rendering.
  • KPI Monitoring: The production dashboard tracked Overall Equipment Effectiveness (OEE) components — Performance (77.8%), Availability (75.3%), and Quality (96.4%) — while the sales dashboard compared forecast vs. actual revenue with heatmap and combo chart visualizations.

Reference: Case study based on Jayadinata, D. H., Pratama, A. T., & Sofianti, T. D. (2024). Development of Performance Monitoring Dashboard for Product Packaging Manufacturer by using Waterfall Methodology. Jurnal Ilmiah Teknik Industri, 23(1), 151–161.

AI-Assisted Data Engineering Pipeline: WeatherAPI to Snowflake

Figure 1.0: Automated data pipeline architecture orchestrated via Express AI, extracting live weather data, applying temperature transformations, and loading into Snowflake.

Project Overview: Developed an end-to-end Extract, Transform, Load (ETL) pipeline utilizing a domain-specific AI agent. The objective was to replace manual boilerplate coding and infrastructure configuration with an intelligent orchestration layer to process live meteorological data.

  • Extraction & AI Orchestration: Utilized the Express AI agent (via nexla.dev) to autonomously configure REST API credentials, infer dynamic JSON schema mappings, and extract live location data from WeatherAPI using natural language prompts.
  • Transformation: Leveraged conversational commands to generate deterministic Python execution logic, dynamically converting incoming Celsius temperature metrics into Fahrenheit mid-stream.
  • Loading & Automation: Bound the pipeline securely to a Snowflake cloud data warehouse and orchestrated a continuous 15-minute CRON schedule to maintain a low-latency historical data ledger.

Reference: Pipeline architecture inspired by the tutorial I built this complete ETL data pipeline with an AI Agent!

Project

Project Industry (PPG): Cloud-Based Inventory Risk Management Pipeline

Group project (Group 7) — proposal, development, and report. Built an automated ELT pipeline utilizing the Medallion Architecture to transition raw supply chain data into analytics-ready insights for inventory risk assessment.

Problem & overview

PPG faced challenges managing inventory data dispersed over several operational datasets, limiting visibility into excess inventory, dead stock, and expired supplies. We designed an automated data pipeline to proactively address seven core business questions regarding inventory health. By detecting Recoverable Assets (RA) and Magna Carta obsolete inventory, the system ultimately allows for the identification of over $1.73M in excess capital and flags downstream impacts on customer fulfillment.

Architecture

Data Flow: Six Operational CSVs → Azure Data Factory (Bronze Layer Ingestion) → Azure Data Lake → Azure Databricks (Silver Layer Cleaning & Business Logic) → Azure SQL Database (Gold Layer Star Schema) → Microsoft Power BI (Interactive Dashboards).

Figure 1.0: Azure Data Orchestration Flow (Bronze/Silver/Gold Layers)

Data Architecture: We implemented a Medallion Architecture, with a Bronze Layer for raw data ingestion, a Silver Layer for incremental cleaning and business rule application, and a Gold Layer structured as a dimensional Star Schema to support efficient analytics and reporting.

Tools & Technologies: Azure Data Factory for orchestration, Azure Data Lake for scalable storage, Azure Databricks for Spark-based transformations, Azure SQL Database for structured storage and querying, and Microsoft Power BI for interactive visualization.

Gold Layer Implementation: I established the ppg_curated database to serve as our analytics-ready Gold Layer. Using automated CREATE EXTERNAL TABLE scripts, the structured Star Schema (utilizing tables like DimMaterial, DimDate, and FactInventoryStatus) was optimized specifically for high-performance querying, acting as the single source of truth for the reporting layer.

My role & contributions

Beyond collaborative pipeline development, I drafted and refined the technical justifications for the data architecture layers, emphasizing the Auditability, Quality Control, and Performance arguments behind our Bronze/Silver/Gold strategy. I also contributed to structuring the dimensional Star Schema in the Azure SQL Database to feed our interactive Power BI logistics dashboards.

Figure 1.0: PPG Case Study Dashboard (built in Microsoft Power BI)

To optimize dashboard performance, I implemented the Power BI data connection using Import Mode. Because the ELT pipeline runs on a scheduled batch-refresh cycle from the Gold Layer, importing the data directly into memory ensures the dashboard can quickly respond to logistics managers' unique business questions with zero query latency, even when filtering complex metrics.

Evidence

Our pipeline successfully consolidated millions of data rows. The Azure Data Factory orchestration seamlessly handled the raw data load, while the Gold Layer SQL analytics accurately generated the $147,565 Magna Carta financial provision breakdown. Additionally, the final one-page consolidated dashboard successfully traces upstream raw material deficits to downstream client impact, utilizing urgency-driven conditional formatting to instantly flag 660 at-risk production units and prioritize critical sales orders.

Figure 1.0: Used Pipeline Tools (Azure Data Factory, Azure Databricks, Azure SQL Database, Power BI)

Challenges & fixes

A key challenge was structuring unstructured, multi-source data without losing historical integrity for auditing purposes. We resolved this by adopting a non-destructive ingestion strategy into the Bronze Layer (keeping native formats) and utilizing Azure Databricks in the Silver Layer to incrementally apply complex business rules, such as automatically flagging unexpired materials with 9+ months of zero consumption as Magna Carta Dead Stock.

Reflection

This project effectively bridged academic data engineering concepts with enterprise-grade solutions. Designing a robust Medallion Architecture in Microsoft Azure demonstrated how proper data layering directly impacts business intelligence, query performance, and proactive financial risk mitigation. Grounding our data strategy in verified industry models like Gorelik's The Enterprise Data Lake proved essential for maintaining a clean, scalable data environment.
