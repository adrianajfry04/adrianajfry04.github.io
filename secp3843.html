Apache Spark ETL Pipeline: Censo Escolar

Project Overview: Developed an end-to-end ETL (Extract, Transform, Load) pipeline to process and analyze the Brazilian School Census dataset. The objective was to transform a large-scale, flat CSV file into a query-optimized multidimensional data model.

  • Extraction & Transformation: Utilized PySpark to ingest the raw data, generate surrogate primary keys, and dynamically construct a Star Schema consisting of a central Fact table and 9 descriptive Dimension tables.
  • Infrastructure & Loading: Containerized the local architecture using Docker and loaded the transformed schema into a PostgreSQL data warehouse.
  • Visualization: Connected Metabase to the relational database to serve interactive business intelligence dashboards and validate the data model.

Reference: Pipeline architecture inspired by the guide Creating a Simple ETL Pipeline With Apache Spark.

Project Industry (PPG): Cloud-Based Inventory Risk Management Pipeline

Group project (Group 7) — proposal, development, and report. Built an automated ELT pipeline utilizing the Medallion Architecture to transition raw supply chain data into analytics-ready insights for inventory risk assessment.

Problem & overview

PPG faced challenges managing inventory data dispersed over several operational datasets, limiting visibility into excess inventory, dead stock, and expired supplies. We designed an automated data pipeline to proactively address seven core business questions regarding inventory health. By detecting Recoverable Assets (RA) and Magna Carta obsolete inventory, the system ultimately allows for the identification of over $1.73M in excess capital and flags downstream impacts on customer fulfillment.

Architecture

Data Flow: Six Operational CSVs → Azure Data Factory (Bronze Layer Ingestion) → Azure Data Lake → Azure Databricks (Silver Layer Cleaning & Business Logic) → Azure SQL Database (Gold Layer Star Schema) → Microsoft Power BI (Interactive Dashboards).

Figure 1.0: Azure Data Orchestration Flow (Bronze/Silver/Gold Layers)

Data Architecture: We implemented a Medallion Architecture, with a Bronze Layer for raw data ingestion, a Silver Layer for incremental cleaning and business rule application, and a Gold Layer structured as a dimensional Star Schema to support efficient analytics and reporting.

Tools & Technologies: Azure Data Factory for orchestration, Azure Data Lake for scalable storage, Azure Databricks for Spark-based transformations, Azure SQL Database for structured storage and querying, and Microsoft Power BI for interactive visualization.

Gold Layer Implementation: I established the ppg_curated database to serve as our analytics-ready Gold Layer. Using automated CREATE EXTERNAL TABLE scripts, the structured Star Schema (utilizing tables like DimMaterial, DimDate, and FactInventoryStatus) was optimized specifically for high-performance querying, acting as the single source of truth for the reporting layer.

My role & contributions

Beyond collaborative pipeline development, I drafted and refined the technical justifications for the data architecture layers, emphasizing the Auditability, Quality Control, and Performance arguments behind our Bronze/Silver/Gold strategy. I also contributed to structuring the dimensional Star Schema in the Azure SQL Database to feed our interactive Power BI logistics dashboards.

Figure 1.0: PPG Case Study Dashboard (built in Microsoft Power BI)

To optimize dashboard performance, I implemented the Power BI data connection using Import Mode. Because the ELT pipeline runs on a scheduled batch-refresh cycle from the Gold Layer, importing the data directly into memory ensures the dashboard can quickly respond to logistics managers' unique business questions with zero query latency, even when filtering complex metrics.

Evidence

Our pipeline successfully consolidated millions of data rows. The Azure Data Factory orchestration seamlessly handled the raw data load, while the Gold Layer SQL analytics accurately generated the $147,565 Magna Carta financial provision breakdown. Additionally, the final one-page consolidated dashboard successfully traces upstream raw material deficits to downstream client impact, utilizing urgency-driven conditional formatting to instantly flag 660 at-risk production units and prioritize critical sales orders.

Figure 1.0: Used Pipeline Tools (Azure Data Factory, Azure Databricks, Azure SQL Database, Power BI)

Challenges & fixes

A key challenge was structuring unstructured, multi-source data without losing historical integrity for auditing purposes. We resolved this by adopting a non-destructive ingestion strategy into the Bronze Layer (keeping native formats) and utilizing Azure Databricks in the Silver Layer to incrementally apply complex business rules, such as automatically flagging unexpired materials with 9+ months of zero consumption as Magna Carta Dead Stock.

Reflection

This project effectively bridged academic data engineering concepts with enterprise-grade solutions. Designing a robust Medallion Architecture in Microsoft Azure demonstrated how proper data layering directly impacts business intelligence, query performance, and proactive financial risk mitigation. Grounding our data strategy in verified industry models like Gorelik's The Enterprise Data Lake proved essential for maintaining a clean, scalable data environment.
